# 蓝绿松石版 · 前端重构开发说明

本项目是「社系毕业晚会解密网站」的**独立副本**，专用于按「蓝绿松石」设计稿重构前端。
与原项目完全分开，互不影响。

## 文件布局

- `设计稿参考_蓝绿松石.html` —— **视觉目标**。自包含静态稿（内联样式 + 假数据，不连后端）。
  双击即可在浏览器查看要做成的样子。重构时照着它来。
- `static/` —— **真正要改的前端**（这是后端实际伺服的内容）：
  - `app.js`（页面渲染逻辑）
  - `styles.css`（配色与样式）
  - `index.html`（外壳）
  - `assets/`（信箱照、简谱图等真实素材）
- `app.py` —— 后端服务器。**纯视觉重构通常不需要改它**；要保持与后端的所有交互不变。
  （例外：本轮「核销逻辑」一项对它做了调整，见文末《核销逻辑》。）
- `data/` —— 运行时自动生成的本机数据库（已被 git 忽略）。

## 重构目标

把 `static/` 里的 `app.js` + `styles.css`（必要时 `index.html`）改写成
`设计稿参考_蓝绿松石.html` 的视觉效果，**同时保持所有答题/抽奖/后台交互和后端数据流不变**。

## 已实现的前端交互与视觉特性

重构已完成，`static/` 现按「蓝绿松石」设计稿呈现。配色全清单见 `视觉配色清单.md`。
以下是改造后新增/调整的交互，便于后续维护：

### 视觉基调
- 松石青主色 + 暖铜唯一焦点 + 青灰中性 + 暖陶红（仅错误态）。
- 标题 / 题面 / 印章用衬线（Songti），正文用无衬线（PingFang）。
- 题卡为四周细边 + 投影的卡片；主线/终章题面为诗性衬线大字，旁枝题面为较小无衬线。

### 页面切换 · 浓雾过渡（`app.js`）
- 触发条件：**页面大标题（`.section-head h1` 文本）发生变化**时（登录↔答题、主线→旁枝/终章、终章→通关、前台↔后台等）。
- 效果：body 层雾幕「聚集→散开」（约 1.15s）+ 内容淡入；同一页内的局部刷新（提交、抽奖、开关弹窗）**不触发**。
- 实现集中在 `setScreen()` → `transitionTo()` / `playPageTransition()`，自动覆盖所有页面，无需逐页接线。
- 注意：内容动画**只用透明度**，不要给 `#app` 加 `filter`/`transform`——否则其内部 `position:fixed` 弹层会以 `#app` 为包含块，在宽屏上错位。雾幕的模糊放在 body 层（`.page-fog`）。

### 回声纸团 / 百宝箱 / 赛博礼物
- 回声出现：纸团展开动效 + 盖印章（取 `echo.highlight` 字）；点「收入百宝箱」会**揉成纸团抛走**再消失。
- 百宝箱：每条回声左侧带印章字；旁枝百宝箱用青绿副色印章。
- 赛博礼物：弹窗出现时在 body 雾层之上**绽放粒子烟花**（`launchFireworks()`，颜色取松石青/暖铜系）。

### 简谱大图翻页（`app.js`）
- 点开《行行重行行》任意一页后，底部出现「‹ 上一页 / 当前/总数 / 下一页 ›」，可**就地翻页不退出**；支持 **←/→ 方向键**。
- 仅简谱走画廊模式（`openImageModal(image, gallery, index)`）；普通题图仍是单图。

## 核销逻辑（前端 `app.js` + 后端 `app.py`）

> 这是唯一**改动了后端**的部分（其余为纯前端重构）。涉及 `preview_redeem_code`、
> `redeem_code`、`send_cyber_gift`、`unredeem_code` 四个函数与前端 `renderAwardDetails`。

工作人员后台「核销兑奖码」输入码后：

- **未核销**：显示参与者信息 + 内联高亮确认块「请确认是否为 26 届毕业生：
  **是 · 核销并发实体礼物** / **不是 · 核销并发赛博礼物**」。
- **已核销（实体或赛博，提示窗口相同）**：显示「**已核销 · 礼物已发送**」+ 对应明细
  「实体礼物已核销 / 赛博礼物已发送：时间，操作人」，并出现「**撤销核销**」按钮。
- 上述「礼物已发送」类提示**统一用红色框**标出（`.redeem-alert`）以示注意。

后端要点：
- `preview` 把「已发赛博礼物」也视作已核销（之前会误判为未核销并再次要求选择）。
- `撤销核销`（`unredeem_code`）会**同时清空实体与赛博两种状态**，撤销后可重新核销并发放。
- 实体核销（`redeem_code`）若发现已发过赛博礼物，会提示需先撤销，避免重复发放。
- 抽奖未完成时不显示核销按钮（沿用原有「请先完成抽奖」拦截）。

如需把内联确认块改成弹窗遮罩形式，可后续再调整（当前为单屏后台的内联确认，已确认保留）。
